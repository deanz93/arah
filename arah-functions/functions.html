Report Lifecycle Functions

onReportCreated
Firestore trigger reports/{id} timeout: 60s maxInstances: 50

Triggered when a new document is created in reports/{id}. Broadcasts to nearby WebSocket clients via Redis pub/sub. Triggers FCM push notifications to users within a 5km radius of the report location.

onVoteUpdated
Firestore trigger reports/{id}/votes/{uid} timeout: 30s maxInstances: 100

Aggregates vote counts on the parent report document using Firestore transactions. Automatically expires (soft-deletes) a report when net downvotes reach −3 or lower.

onReportExpired
Scheduled every 5 min timeout: 120s maxInstances: 1

Scans the active reports index for entries past their TTL. Removes them from the active geospatial index, archives the full document to reports_history/{id}, and emits a report:expired WebSocket event.

moderateReport
HTTP callable admin only timeout: 30s maxInstances: 10

Admin-callable function to approve, reject, or escalate a report with a reason string. Restricted to users with the admin custom claim. Updates report status and triggers downstream notifications.

User & Gamification

onReportVerified
Firestore trigger reports/{id} timeout: 30s maxInstances: 50

Fires when a report's status field transitions to verified. Awards XP to the submitting user and evaluates badge unlock thresholds. Updates users/{uid}/stats.

calculateStreak
Scheduled daily 00:00 MYT timeout: 300s maxInstances: 1

Nightly job that computes contribution streaks for all active users. Resets streaks for users with no activity in the past 24h. Runs at midnight Malaysian time (UTC+8).

getUserStats
HTTP callable timeout: 15s maxInstances: 100

Returns the calling user's leaderboard position, XP total, current streak, and full badge list. Reads from pre-computed stats in users/{uid}/stats for sub-100ms response.

Notifications

sendFloodAlert
HTTP callable admin only timeout: 60s maxInstances: 5

Admin-only callable that broadcasts an FCM emergency alert to all users whose locationState matches the specified flood zone ID. Uses FCM topic messaging for scale.

notifyNearbyDrivers
Firestore trigger reports/{id} timeout: 30s maxInstances: 50

FCM batch send to active drivers within 5km of a newly created or verified incident. Queries geohash index to find nearby FCM tokens, batches up to 500 tokens per FCM multicast call.

sendPrayerReminder
Pub/Sub prayer-reminders topic timeout: 60s maxInstances: 10

Subscribed to the prayer-reminders Pub/Sub topic. Sends a pre-prayer FCM notification to users who have opted in, segmented by Malaysian state (each state has different prayer times).

Data Operations

weeklyAnalyticsRollup
Scheduled every Sunday 01:00 MYT timeout: 540s maxInstances: 1

Weekly aggregation job that rolls up report statistics per Malaysian state and per report type into analytics/weekly/{YYYY-WW}. Powers the admin dashboard charts.

cleanExpiredData
Scheduled daily 03:00 MYT timeout: 540s maxInstances: 1

Purges GPS ping documents older than 24 hours from gps_pings and archived report documents older than 7 days from reports_history. PDPA data retention compliance.

exportUserData
HTTP callable timeout: 300s maxInstances: 20

PDPA Article 30 compliant data export. Collects all user data (profile, reports, votes, GPS history) into a JSON ZIP file, uploads to Cloud Storage, and returns a signed download URL valid for 24 hours.

Admin & Moderation

onUserDeleted
Auth trigger user.delete timeout: 300s maxInstances: 20

Triggered by Firebase Auth user deletion event. Performs PDPA-compliant cascade delete: removes users/{uid}, all authored reports, all votes, GPS ping history, and analytics associations. Uses batched writes for atomicity.

generateModerationReport
Scheduled every Monday 08:00 MYT timeout: 120s maxInstances: 1

Weekly moderator summary emailed to admin addresses. Includes: total reports submitted, verified, rejected, and escalated in the past week, plus top reporting users and flagged content.
