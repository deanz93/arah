Contributing to arah-functions

Development setup, branch conventions, and PR checklist

Environment Setup

Prerequisites
Node.js ≥ 22.0.0 (LTS)
Firebase CLI ≥ 13.0.0
Java JDK ≥ 11 (for emulators)
Clone & Install
git clone https://github.com/deanz93/arah-functions.git
cd arah-functions
npm ci
Start Emulator Suite
firebase emulators:start
# Starts: Functions (5001), Firestore (8080), Auth (9099), Pub/Sub (8085)
# Emulator UI: http://localhost:4000
Run Tests Against Emulator
npm run test          # Vitest unit tests
npm run test:emulator # Integration tests against emulator suite
npm run typecheck     # tsc --noEmit
npm run lint          # ESLint
Deploy (after CI passes)
firebase deploy --only functions              # All functions
firebase deploy --only functions:onReportCreated  # Single function

Branch & Commit Conventions

Branch Format
feature/FN-NNN-short-description
fix/FN-NNN-short-description
chore/description
✓ feature/FN-001-on-report-created
✓ fix/FN-004-vote-aggregation-race
✗ my-function (no story ID)
Commit Format
feat(functions): add onReportCreated trigger (FN-001)
fix(functions): fix vote race condition (FN-004)
test(functions): add emulator test for sendFloodAlert
chore: update firebase-admin to 12.x
Scope: always functions for function changes.
Include story ID in parentheses when applicable.

Skill Files

Skill files in .claude/skills/ guide AI-assisted development. Read them before working on related code.

function.md

Conventions for new Cloud Functions v2: trigger patterns, timeout/maxInstances config, idempotency requirements, and error handling. Covers Firestore, Scheduled, Callable, Auth, and Pub/Sub trigger types.

test.md

Emulator-backed integration test patterns: how to seed Firestore data, trigger functions programmatically, assert side-effects, and clean up after tests. Unit test mocking patterns for Firebase Admin.

pr.md

Pull request checklist, description template, review criteria, and story definition-of-done gates specific to Cloud Functions development.

Pre-PR Checklist

Cloud Functions v2 Notes

Always use v2 imports
// Correct — v2
import { onDocumentCreated } from 'firebase-functions/v2/firestore'
import { onCall } from 'firebase-functions/v2/https'
import { onSchedule } from 'firebase-functions/v2/scheduler'

// Wrong — v1
import * as functions from 'firebase-functions'
Function config pattern
export const onReportCreated = onDocumentCreated(
  {
    document: 'reports/{id}',
    timeoutSeconds: 60,
    maxInstances: 50,
    region: 'asia-southeast1',
  },
  async (event) => { ... }
)
