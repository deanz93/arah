Contributing

How to contribute to arah-geocoding — setup, branch format, skill files, and PR checklist

Getting Started

Prerequisites

  • ✓ Docker Compose v2
  • ✓ Python 3.11+ (for test scripts)
  • ✓ pytest + pytest-httpx
  • ✓ k6 (for load testing)
  • ✓ AWS CLI (for EBS snapshot ops)

Local Setup

git clone https://github.com/deanz93/arah-geocoding.git
cd arah-geocoding
# Start pre-built Nominatim image
docker compose up nominatim
# Verify geocoding works
curl 'http://localhost:7070/search?\
  q=KLCC&countrycodes=my&format=jsonv2'

Branch & Commit Format

Branch naming

feature/GEO-NNN-short-description

Examples:

feature/GEO-010-bm-name-search
feature/GEO-025-cache-warming
feature/GEO-026-bm-timeout-error

Commit convention

feat(geocoding): short description

Conventional Commits prefixes:

  • feat(geocoding): — new feature
  • fix(geocoding): — bug fix
  • chore(geocoding): — maintenance
  • docs(geocoding): — documentation
  • test(geocoding): — tests only

Skill Files

test.md

Test runbook — how to run pytest integration tests and validate_coverage.py for all 16 Malaysian states.

import.md

Data import guide — Nominatim first-time import, incremental updates, EBS snapshot restore, and validation gate.

pr.md

PR template and review checklist — what reviewers look for in geocoding API, data, and deployment PRs.

Pre-PR Checklist

All items must pass before opening a pull request. CI enforces these gates automatically.

pytest integration tests pass

pytest tests/integration/ -v — all test cases green

validate_coverage.py — all 16 states pass

python scripts/validate_coverage.py — all state postcode prefixes return results

k6 p95 < 300ms

50 VU mixed /search + /reverse scenario, p95 latency gate at 300ms

countrycodes=my enforced

Integration test confirms requests without countrycodes=my return 400 or are coerced

No PII in logs

Log audit script confirms only query_hash is logged, not raw query strings

EBS snapshot before any import

If the PR modifies data import: confirm EBS snapshot taken and snapshot ID logged

PR Workflow

# 1. Create feature branch
git checkout -b feature/GEO-NNN-description

# 2. Make changes, then run checks
pytest tests/integration/ -v
python scripts/validate_coverage.py
k6 run scripts/load-test.js

# 3. Commit with conventional format
git commit -m "feat(geocoding): BM name search (GEO-010)"

# 4. Push and open PR
git push origin feature/GEO-NNN-description
gh pr create --fill

PDPA Compliance Note

This service handles user location queries. Never log raw query strings — only SHA-256 hashes. Never persist user coordinates beyond the current request. If adding a new log line, run the PII audit script before committing: python scripts/audit_logs.py
