Contributing

How to contribute to arah-tile-server — setup, branch format, skill files, and PR checklist

Getting Started

Prerequisites

  • ✓ Docker Desktop / Docker Engine
  • ✓ Node.js 24 LTS
  • ✓ Java 21 (for local Planetiler)
  • ✓ k6 (for load testing)
  • ✓ Playwright (for visual regression)

Local Setup

git clone https://github.com/deanz93/arah-tile-server.git
cd arah-tile-server
# Build and run locally
docker build -t arah-tile-server:local .
docker run -p 8080:8080 \
  -v $(pwd)/data:/data \
  arah-tile-server:local
# Verify
curl http://localhost:8080/health

Branch & Commit Format

Branch naming

feature/TILE-NNN-short-description

Examples:

feature/TILE-012-bilingual-labels
feature/TILE-028-jawi-rtl-layer
feature/TILE-021-k6-load-test

Commit convention

feat(tiles): short description

Conventional Commits prefixes:

  • feat(tiles): — new feature
  • fix(tiles): — bug fix
  • chore(tiles): — maintenance
  • docs(tiles): — documentation
  • test(tiles): — tests only

Skill Files

style.md

MapLibre style authoring guide — colour tokens, layer naming conventions, zoom ranges, and Jawi RTL requirements.

test.md

Testing runbook — how to run Playwright visual regression tests and k6 scenarios against a local TileServer instance.

pr.md

PR template and review checklist — what reviewers look for in tile style, pipeline, and deployment PRs.

Pre-PR Checklist

All items must pass before opening a pull request. CI enforces these gates automatically.

Docker build passes

docker build -t arah-tile-server:local . exits 0

Playwright visual regression tests pass

Screenshot diffs within 0.5% threshold for day and night styles

k6 p95 < 200ms

Tile endpoint p95 latency under 200ms at 100 VU sustained load

pmtiles verify passes

pmtiles verify malaysia.pmtiles — checksum and header validation

Docker uid 1001

Container process runs as non-root uid 1001 — verified by docker run --user check

Both day + night styles render

/styles/arah-day/style.json and /styles/arah-night/style.json return valid MapLibre JSON

Jawi layer present

place-label-jawi layer present in style JSON with visibility set to none

PR Workflow

# 1. Create feature branch
git checkout -b feature/TILE-NNN-description

# 2. Make your changes, then run checks
docker build -t arah-tile-server:local .
npx playwright test --update-snapshots  # first run only
k6 run scripts/load-test.js

# 3. Commit with conventional format
git commit -m "feat(tiles): add jawi label layer (TILE-028)"

# 4. Push and open PR
git push origin feature/TILE-NNN-description
gh pr create --fill
