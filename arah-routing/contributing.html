Contributing

Contributing to arah-routing

Profile authoring, test conventions, and the pre-PR checklist for routing contributors.

Getting Started

1
Fork & Clone

Fork the repo and clone locally. You need Docker to build and test the engine.

2
Build Locally

Run docker build -t arah-routing:local . and start the engine.

3
Open a PR

Run the smoke tests and performance gate before requesting review.

Branch Naming

All branches must be tied to a story ID in the format RTE-NNN.

# Format
feature/RTE-NNN-short-description

# Examples
feature/RTE-009-auto-costing-profile
feature/RTE-023-toll-plaza-database
feature/RTE-031-blue-green-tile-swap

Commit Format

Follow Conventional Commits with the routing scope.

# Format
feat(routing): short description of what was done

# Examples
feat(routing): add auto_toll_free costing profile
fix(routing): correct motorcycle speed limits on federal roads
chore(routing): pin Valhalla to v3.4.0
test(routing): add k6 benchmark for KL-Penang route
Prefix When to use
feat(routing):New profile, pipeline script, API feature
fix(routing):Incorrect speed limits, broken smoke test, pipeline failure
chore(routing):Valhalla version pins, dependency updates
test(routing):k6 scripts, smoke tests, profile validation tests
docs(routing):Profile docs, runbook updates, API examples

Pre-PR Checklist

Run all checks locally before pushing. CI enforces them, but local failures save round-trips.

Docker build passes
docker build -t arah-routing:local .
Smoke tests pass
./scripts/smoke.sh http://localhost:8002

Sends 5 test routes (KL → Penang, KL → JB, Petaling Jaya local, motorcycle, pedestrian) and checks status 200 + non-empty geometry.

Python profile validation tests pass
python -m pytest tests/profiles/ -v

Validates each profile JSON schema, speed limit ranges, and that toll avoidance flags are mutually consistent.

p95 latency < 500ms maintained
k6 run scripts/benchmark.js --env HOST=http://localhost:8002

Runs 200 virtual users for 60 seconds. p95 must stay below 500ms. CI fails the PR if this threshold is breached.

Memory under 5GB at peak load

Monitor RSS during the k6 run. The Valhalla graph is ~3.8GB; overhead must not push RSS above 5GB. Document measured peak in the PR description.

Valhalla version pinned

The Dockerfile must use an exact Valhalla version tag (e.g. valhalla/valhalla:3.4.0). :latest is not permitted in the Dockerfile.

Writing a New Routing Profile

Profiles live in profiles/. Each is a JSON file passed to Valhalla at startup.

{
  "costing": "auto",
  "costing_options": {
    "auto": {
      "use_tolls": 0.5,          // 0 = avoid, 1 = prefer
      "use_highways": 1.0,
      "use_ferry": 0.5,
      "top_speed": 110           // Malaysia NSE speed limit
    }
  },
  "directions_options": {
    "language": "ms-MY"          // Bahasa Malaysia
  }
}
Do
  • ▸ Set language: "ms-MY" on all profiles
  • ▸ Use top_speed: 110 for highway profiles
  • ▸ Add a validation test in tests/profiles/
Don't
  • ▸ Use top_speed above 110 (NSE limit)
  • ▸ Set use_highways: 0 on car profiles (breaks ETA)
  • ▸ Ship a profile without a smoke test route
