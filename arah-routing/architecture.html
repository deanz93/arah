How Valhalla Works

Valhalla is an open-source C++ routing engine. On startup it loads a pre-built graph of Malaysian roads into memory (~3.8GB RAM). Each route request runs a bidirectional A* search over this in-memory graph and returns a response in under 500ms — even for 350km cross-peninsula routes. The graph is never re-read from disk at query time, which is the source of its low latency.

OSM Data Pipeline

The pipeline runs monthly via a GitHub Actions workflow. The tile swap is atomic — pods continue serving old tiles until the new graph is fully built and validated.

Geofabrik (monthly)
   │ malaysia-latest.osm.pbf
   ▼
Valhalla tile builder (Docker)
   │ valhalla_tiles/ (~3.8GB)
   ▼
EFS shared storage
   │ (atomic mv, ReadOnlyMany)
   ▼
Valhalla K8s pods (2 replicas)
   │
   ▼
REST API → arah-api (proxy)

API Endpoints

All endpoints are proxied through arah-api. The Valhalla pod is not exposed directly to the internet.

POST
/route

Point-to-point routing with up to 5 waypoints. Returns turn-by-turn manoeuvres, distance, duration, and polyline geometry.

POST
/optimized_route

Travelling Salesman Problem (TSP) solver for delivery routing. Finds the shortest ordering of multiple stops.

POST
/isochrone

Reachability polygon: generates drive-time zones (e.g. "all areas reachable within 30 minutes from KL Sentral").

GET
/status

Engine health check. Returns Valhalla version, graph build date, and tile set version. Used by Kubernetes liveness probes.
