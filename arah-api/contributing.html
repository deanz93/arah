Contributing to arah-api

Development setup, branch conventions, and PR checklist

Environment Setup

Prerequisites
Node.js ≥ 22.0.0 (LTS)
Redis 7.1+ (local or Docker)
Firebase Service account JSON
Clone & Install
git clone https://github.com/deanz93/arah-api.git
cd arah-api
npm ci
Environment Variables
cp .env.example .env
# Fill in:
# FIREBASE_SERVICE_ACCOUNT_PATH=./service-account.json
# REDIS_URL=redis://localhost:6379
# PORT=3000
# NODE_ENV=development
Run Development Server
npm run dev        # http://localhost:3000 with hot reload
npm run test       # Vitest unit + integration tests
npm run typecheck  # tsc --noEmit
npm run lint       # ESLint

Branch & Commit Conventions

Branch Format
feature/API-NNN-short-description
fix/API-NNN-short-description
chore/description
✓ feature/API-009-post-reports-endpoint
✓ fix/API-012-vote-idempotency
✗ my-feature (no story ID)
Commit Format
feat(api): add POST /reports endpoint (API-009)
fix(api): correct vote idempotency check (API-012)
test(api): add integration test for /auth/me
chore: update dependencies
Allowed scopes: api, ws, auth, db, ci
Include story ID in parentheses when applicable.

Skill Files

Skill files in .claude/skills/ guide AI-assisted development. Read them before working on related code.

route.md

Conventions for adding new Fastify routes: plugin structure, Zod schema, response envelope, error codes, auth guard placement.

test.md

Integration test patterns using Vitest + Supertest: test database seeding, Firebase auth mocking, Redis test instance setup.

pr.md

Pull request checklist, description template, review criteria, and story definition-of-done gates.

Code Style Notes

ESM-first
All imports use .js extension (ESM Node.js requirement). No CommonJS require().
import { reportSchema } from './schemas/report.js'
Fastify Plugin Pattern
Routes wrapped in fastify.register() plugin functions with encapsulation. No top-level route registrations.
export default fp(async (fastify) => {
  fastify.post('/reports', { schema }, handler)
})
