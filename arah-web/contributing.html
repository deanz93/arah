Contributing

arah-web is open for contributors. Set up your Next.js dev environment, pick a story, and open a pull request.

GitHub Repository ↗ CLAUDE.md ↗ Stories (04-stories.md) ↗

Environment Setup

1
Fork the repository

Click Fork on GitHub, then clone your fork locally.

git clone https://github.com/YOUR_USERNAME/arah-web.git
cd arah-web
2
Install dependencies

Use npm ci for reproducible installs matching the lock file exactly.

npm ci
3
Start the dev server

Next.js 14 dev server with Turbopack — hot reload in under 1s.

npm run dev
# → http://localhost:3000
4
Run tests

Unit tests with Vitest + React Testing Library, E2E with Playwright.

# Unit tests (watch mode)
npm run test

# E2E tests
npx playwright test
5
Pick a story and build

Open docs/bmad/04-stories.md, find an MVP story without an assigned contributor, comment on the issue, then create your branch.

🌿 Branch Format 

feature/WEB-NNN-short-description
feature/WEB-001-admin-login
feature/WEB-011-report-queue
feature/WEB-035-flood-zone-map

📝 Commit Format

Conventional Commits — required for automated changelogs.

feat(web): add admin login page

Implements WEB-001. Firebase Auth with
Google SSO, redirects to /dashboard.
feat — new feature
fix — bug fix
chore — deps, config, cleanup

Claude Skills

Three slash commands in .claude/skills/ to speed up your workflow in Claude Code.

/component
component.md

Scaffolds a new shadcn/ui-based React component with Tailwind utilities, dark mode variants, TypeScript props interface, and ARIA attributes.

/test
test.md

Generates Vitest + React Testing Library test files covering render, user interaction, and error state — one file per component following the project naming convention.

/pr
pr.md

Runs the full pre-PR checklist, summarises changes in the BMAD story format, and opens a GitHub pull request with correct labels and the linked WEB story ID.

Pre-PR Checklist

Commands

npm run typecheck
0 errors
npm run lint
0 warnings
npm run test:run
all pass
npm run build
no errors

Code Quality

  • ✓ Tailwind only — no custom CSS files (.css, .scss, .module.css)
  • ✓ Use shadcn/ui components — don't build from raw HTML
  • ✓ Dark mode dark: variants on all colour utilities
  • ✓ Mobile responsive at 375px viewport width
  • ✓ Story ID referenced in PR title and description
🌟

Good First Issues

All stories tagged MVP in docs/bmad/04-stories.md are good starting points. WEB-001 (admin login), WEB-002 (dashboard overview), and WEB-011 (report queue) are the most self-contained for first-time contributors.
