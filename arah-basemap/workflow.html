Update Workflow

Scheduled rebuild cadence, nightly OSM diff patching, manual hotfix procedure, and rollback steps.

Weekly Full Rebuild

Fetch OSM PBF
Download malaysia-singapore-brunei-latest.osm.pbf from Geofabrik. Verify md5.
scripts/fetch-osm.sh
Generate 2D tiles  ~25 min
Full Planetiler run on 8-core machine with 8 GB heap.
scripts/build-2d.sh
Rebuild terrain (if version changed)
Only runs if Copernicus DEM source has a newer version.
scripts/build-terrain.sh --if-newer
Validate
Tile count check + style-spec validation. Fails CI if count drops >5%.
npm test
pmtiles verify dist/malaysia-2d.pmtiles
Upload to S3 + notify
Upload PMTiles, set cache headers, post to Slack #maps-ops.
scripts/upload.sh --layer 2d
# → s3://arah-tiles-prod/2d/malaysia-2d.pmtiles
# → Slack: "Weekly tile build complete ✅ (2.8 GB)"

Nightly Diff Patch

Keeps roads and POIs fresh without a full rebuild. Takes ~8 min.

scripts/diff-update.sh
# 1. Download OSM minutely diffs since last run
# 2. Apply diffs with osmupdate → malaysia-updated.osm.pbf
# 3. Run Planetiler only on changed tile bounding boxes
# 4. Merge new tiles into existing PMTiles archive
# 5. Upload delta tiles to S3
# Cost: ~$0.02/night (EC2 t3.xlarge, 8 min)

Nightly diff adds ~50 MB to the archive. The weekly full rebuild compacts it back down.

Manual Hotfix

Use when a specific area has critical data errors — wrong road name, missing hospital, incorrect routing.

# Rebuild tiles for a city-district bounding box only (~5 min)
scripts/hotfix.sh --bbox "3.05,101.55,3.25,101.75"

# Bbox format: "lat_min,lon_min,lat_max,lon_max"
# Examples:
# KL centre:   "3.12,101.65,3.18,101.72"
# Georgetown:  "5.39,100.33,5.44,100.37"
# JB city:     "1.44,103.71,1.50,103.78"

Rollback

By date

scripts/rollback.sh --to 2026-07-13
# Copies previous week's PMTiles from
# S3 versioned bucket
# Invalidates CloudFront /tiles/*

Retention

S3 versioning keeps 4 weekly builds
After 28 days, old versions are deleted by lifecycle rule
CloudFront invalidation takes ~30 seconds globally

Adding a New Layer

1
Add OSM tag mapping to profiles/malaysia-full.java
2
Add layer definition and paint rules to styles/arah-2d.json
3
Run npm run validate-style — must pass before continuing
4
Test with a small bbox: scripts/build-2d.sh --layer <name> --bbox "3.1,101.6,3.2,101.7"
5
Run npm test — all tests must pass
6
Open a PR → merge → next weekly build picks it up automatically
